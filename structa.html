Structa

A productivity tool designed to reduce cognitive overload, not increase it.

Problem Statement

Modern productivity tools overwhelm users with excessive options, notifications, and complex hierarchies, leading to stress, decision fatigue, and abandonment.

User Persona

Sarah, 26 — remote worker / student. Struggles with long to-do lists, prioritization, and mental fatigue caused by task overload.

Objective

Reduce cognitive overload during task organization by simplifying hierarchy and minimizing decision points.

  • ↓ Task creation time by 30%
  • ≤ 5 visible choices per screen
  • ↑ User clarity score from 5/10 to 8/10

UX Hypothesis

If we reduce visible choices and guide users step by step, they will feel less overwhelmed and more in control of their workload.

UX Strategy

  • One primary focus area per screen
  • Progressive disclosure of tasks
  • Clear hierarchy using spacing and typography
  • Removal of non-essential UI elements

User Flow

Open app → See one focus → Add task → Complete task → Visual feedback → Exit

Key Learnings

Cognitive overload is a design problem, not a user problem. Reducing information leads to better decisions and sustained use.
